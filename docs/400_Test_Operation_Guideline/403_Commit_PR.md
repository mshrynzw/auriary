# テスト運用ガイドライン：コミット・PR規約

## 3. Pull Request / Commit 規約

### 3.1 コミットメッセージ

テスト関連のコミットは以下の形式を使用：

```
test: 日記作成の単体テストを追加
test(integration): 日記APIの結合テストを追加
test(e2e): 日記作成フローのE2Eテストを追加
fix(test): 日記テストのバグを修正
```

### 3.2 プルリクエストの説明

プルリクエストには以下を含める：

```markdown
## 変更内容
日記作成機能を実装しました。

## テスト
- [x] 単体テストを追加
- [x] 結合テストを追加
- [x] E2Eテストを追加
- [x] すべてのテストが通過

## テストカバレッジ
- 単体テスト: 85%
- 結合テスト: 主要APIをカバー
- E2Eテスト: 日記作成フローをカバー
```
